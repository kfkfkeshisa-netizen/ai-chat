# Codex作業ルール

このリポジトリでCodexが作業する場合は、実装変更だけでなく、必要に応じて要件定義書・詳細設計書・READMEも最新化する。

特に以下を守る。

- 要件や仕様に影響する変更をした場合、関連するMarkdown文書へ反映する。
- 実装だけを進めて、設計書を古いままにしない。
- 実メール、ログ、サンプルデータ、認証情報など機微情報はGit管理対象にしない。
- コミット・pushはユーザーの明示指示がある場合のみ行う。
- 作業前後にローカル変更の状態を確認し、ユーザーの未コミット変更を勝手に戻さない。
- テストや確認を実行した場合は、結果を簡潔に報告する。
